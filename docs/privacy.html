Privacy Policy

Last updated: April 23, 2026

Meridian is a Chrome extension that acts as a persistent AI layer across the web. It summarizes your browsing, answers questions about what you've read, and helps you make better decisions. This policy explains exactly what data Meridian touches, where it lives, and what we do — and do not — do with it.

The short version: Meridian stores everything locally on your device. We operate no servers, run no analytics, and never see your data. Your Claude API key calls Anthropic directly from your browser. You can wipe every byte Meridian has captured at any time with one click.

1. What Meridian captures

When you visit a web page and stay on it for roughly 15 seconds, Meridian's content script extracts:

  • Page URL and title
  • Main visible text content (with scripts, forms, inputs, and sensitive tags stripped)
  • Timestamp of the visit

This content is processed through a credential-redaction pass (see §4) and stored in an IndexedDB database named MeridianDB inside your browser profile. It never leaves your machine except when you explicitly ask Meridian a question — at which point it is sent to Anthropic's Claude API, using your API key, directly from your browser.

2. What Meridian does NOT capture

  • Privacy Zones are hard-blocked. Banking, payment, healthcare, government, authentication, password-manager, dating, and adult sites are never captured. This is enforced in two places: the content script refuses to send, and the service worker refuses to store. Examples of auto-skipped hosts: paypal.com, stripe.com, chase.com, mychart.com, accounts.google.com, 1password.com, and any URL path matching /login, /signin, /oauth, /2fa, /checkout, /payments, or /billing.
  • Paused sites you add yourself in Settings → Privacy & Safety are also skipped.
  • Form fields, password boxes, and input values are never read — the extractor ignores <input>, <textarea>, and <form> DOM nodes entirely.
  • Incognito windows are not captured unless you explicitly grant Meridian incognito access in chrome://extensions.
  • Browser history outside of Meridian. We do not read your chrome.history API or your bookmarks unless you use the History tab, and even then only summaries stay in local IndexedDB.

3. Where your data lives

  • IndexedDB (MeridianDB, MeridianHistoryDB) — captured pages, conversations, research plans, synthesis results.
  • chrome.storage.local — your Claude API key, voice language preference, active mode, paused-site list, tutorial-seen flag.

Nothing is synced to Google's cloud. Nothing is uploaded to any server we control. We do not operate a server.

4. Credential redaction

Before any captured text is stored, Meridian runs a redaction pass that replaces the following patterns with labeled placeholders (e.g. [REDACTED_ANTHROPIC_KEY]):

  • Anthropic API keys (sk-ant-…)
  • OpenAI API keys (sk-…)
  • GitHub personal access tokens (ghp_…)
  • Slack tokens (xoxb-…, xoxp-…)
  • AWS access keys (AKIA…)
  • JWT tokens (eyJ…)
  • Credit card numbers (validated with the Luhn checksum to reduce false positives)
  • US Social Security Numbers
  • password:, api_key=, Bearer <token> inline patterns

The raw values are never written to disk. If a page accidentally contains a secret in its body text, Meridian replaces it before persisting.

5. Third parties

Meridian makes network requests to exactly one third party:

  • Anthropic (api.anthropic.com) — for Claude AI completions, using your own API key, originating from your browser. Anthropic's data handling is governed by Anthropic's Privacy Policy. At the time of writing, Anthropic does not use API inputs to train models.

Meridian does not use analytics (no Google Analytics, no Mixpanel, no Sentry), does not load third-party scripts or fonts, does not embed tracking pixels, and does not contact any telemetry endpoint.

6. Prompt injection defense

When page text is sent to Claude for summarization or answering, Meridian wraps it in sentinel markers (<<<MERIDIAN_UNTRUSTED_CONTEXT>>>) and instructs Claude to treat the wrapped content as data, not instructions. This is a mitigation against malicious pages that try to smuggle commands into the model. It is not a perfect defense, and you should remain skeptical of AI output about untrusted pages.

7. Your rights and controls

  • Pause on this site — Settings → Privacy & Safety. Stops capture on the current host.
  • Clear conversation memory — Brain tab → Clear Memory. Drops chat history.
  • Erase all Meridian memory — Settings → Danger zone. Wipes both IndexedDB databases completely. Your API key and preferences stay, but every captured page, conversation, and synthesis is gone.
  • Uninstall — removing the extension deletes its entire storage, per Chrome's extension lifecycle.

Because Meridian stores data locally and we have no servers, there is nothing for us to delete on our side — uninstall is complete deletion.

8. Children's privacy

Meridian is not directed at children under 13 and we do not knowingly collect data from them. Meridian collects no data centrally in any case.

9. Changes to this policy

If this policy changes materially, we will update the "Last updated" date at the top and note the change in the extension's release notes on GitHub. Continued use of the extension after a change constitutes acceptance.

10. Contact

Questions, concerns, or security reports: meridian.extension@proton.me. Source code and issue tracker: github.com/Dev14Ayoub/Meridian.
